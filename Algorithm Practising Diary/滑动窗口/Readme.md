滑动窗口算法练习心得
 
滑动窗口的核心是用左右指针划定区间，通过动态调整窗口范围解决子串/子数组问题，能将暴力枚举的O(n²)甚至O(n³)复杂度降至O(n)，效率显著。
 
其适用场景集中在子串/子数组的范围查询：比如“长度最小的子数组”“乘积小于k的子数组”这类求“满足条件的区间”问题；或是“无重复字符的最长子串”“最多K个重复元素的最长子数组”这类“限制条件下的范围极值”问题。
 
关键在于把握“窗口移动逻辑”：
 
- 右指针通常负责“扩张窗口”，逐个纳入元素，直到满足条件（或不满足）；
- 左指针负责“收缩窗口”，当窗口不符合要求（或需优化结果）时右移，同时更新窗口内信息。
 
练习中需注意细节：比如“最小覆盖子串”需记录字符频次，收缩时要避免漏判；“统计最大元素出现至少K次的子数组”需结合元素出现位置精准调整窗口。此外，部分题目需灵活选择“窗口是否固定大小”，像“找到最长的半重复子字符串”就需根据条件动态伸缩，而非机械移动。
